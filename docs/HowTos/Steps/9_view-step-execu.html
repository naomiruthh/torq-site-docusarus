When a step is executed in a workflow, the execution is recorded in the Execution Log. The Execution Log is an easy-to-read table of the step's historic executions (for both on-demand execution and as part of a running workflow). Successful step execution is indicated with a green checkmark, and failed step execution is marked with a red checkmark. Both successful and failed step executions are recorded in the log. When a step fails, the step status code will give more information about why the step failed.

Each step execution receives an entry in the log, including the execution ID, the entity that executed the step, the time the step was executed, and the step JSON (inputs and outputs). Subsequent steps can access the event JSON using $.{step_name}, which is the lowercase/snake-case name of the step. For example, the step Add a user would appear in the workflow context as $.add_a_user.
