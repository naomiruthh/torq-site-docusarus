Use the workflow version history to view and restore previous workflow versions or to compare two sequential versions. If there is an existing draft version, it is also shown for comparison.

A version history entry is created for published workflows or current drafts only.

View version history

  1. Go to the Workflows page.
  2. Open the workflow you want to view the version history of.
  3. Open the menu and select Version history. 
  4. Click on a version to view it. Steps that have been changed will be highlighted in blue (modifications) or green (additions). Deleted steps will be listed in a dropdown at the top of the page (in a red window).

The information listed for every version: 

  • The date that the version was created
  • The user who published the workflow or created the draft (if there is a current draft).
  • Version description provided upon publishing. 

View modified fields

Modified steps will have their changed fields highlighted. 

Compare previous values by clicking on the View previous version icon.


üóíÔ∏è Note
Modification information for annotations, trigger parameters, or workflow settings will not be shown.



There are several actions supported by accessing each version's individual menu (upon hovering over the version in the sidebar), such as: 

Rename versions

Give the version a meaningful name to easily locate it in the version history.

Save as workflow

Save an old version as a new individual workflow to continue working separately. This option does not change the current workflow draft.

Restore version

Restoring an old version overwrites the current draft. Any unpublished changes will be lost.
