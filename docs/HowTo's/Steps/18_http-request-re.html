You can set HTTP request steps to automatically retry the request on failures based on the specific response status code. This is especially helpful for dealing with momentary in-availabilities, which would otherwise require the implementation of difficult retry patterns.

The retry mechanism is based on the exponential backoff algorithm with a maximum retry delay of 50 seconds.

Supported steps

You can set the following steps to retry:

  • Send an HTTP Request step
  • Any step that you can Switch to HTTP Mode

How to use

  1. Make sure you're using the Send an HTTP request step version 4.1.5 or newer.
  2. Add the optional parameter RETRY_ON_STATUS.
  3. Specify the HTTP response statuses to retry on. You can specify integers or ranges of integers.
  4.  The step will attempt retries using the exponential backoff algorithm as long as the status code is one of the codes specified. The retries will stop once a different status code is received or a retry delay of 50 seconds is reached.
  5. Control the maximum number of retries by using the optional MAX_RETRIES parameter. If this parameter isn't used, the default behavior is to retry until a retry delay of 50 seconds is reached.
  6. Control the delay from the first step failure until the first retry by using the optional RETRY_DELAY parameter. The value you specify will be the base for the exponential backoff algorithm calculation.
